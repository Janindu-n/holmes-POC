## 2025-05-14 - Optimized scroll performance and image loading
**Learning:** In Next.js/React applications with heavy landing pages, using React state for scroll-based animations (like parallax) causes the entire component tree to re-render on every scroll event, leading to significant main-thread jank. Replacing state with `useRef` and `requestAnimationFrame` for direct DOM manipulation eliminates these re-renders (from 20+ per scroll sequence to 0). Additionally, native `<img>` tags with `fetchPriority="high"` and `loading="lazy"` provide more granular control over LCP than CSS background images.
**Action:** Use `requestAnimationFrame` and direct DOM updates via `useRef` for scroll-linked animations. Always check if critical-path images can be optimized with `fetchPriority`.
