## 2025-05-14 - Optimizing Scroll-Linked Animations in Next.js
**Learning:** In Next.js/React, using `useState` for high-frequency events like scroll causes the entire component to re-render on every frame, leading to significant performance degradation (especially on large pages like the landing page). Bypassing React state and using `useRef` with `requestAnimationFrame` and a "ticking" flag allows for 60fps animations by manipulating the DOM directly without triggering React's reconciliation.
**Action:** For any parallax or scroll-linked effects, prioritize direct DOM manipulation over React state. Use `requestAnimationFrame` to sync with the browser's refresh rate and a `ticking` flag to prevent redundant calls within a single frame. Ensure an initial sync call on mount to handle the current scroll position.
